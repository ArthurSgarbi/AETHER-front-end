## 📋 Sobre o Projeto
O **AETHER** é um ecossistema de software projetado para erradicar o "apagão de dados" ocorrido em centros de comando tático durante desastres climáticos severos. Nossa missão é garantir que agências de gerenciamento de crise mantenham a consciência situacional mesmo quando a infraestrutura urbana de telecomunicações colapsa.

A arquitetura utiliza o princípio de *Edge Computing* para transferir a responsabilidade de monitoramento para as bordas da rede, retendo dados cruciais em ambientes isolados e emitindo alertas de emergência via radiofrequência para impedir que equipes de resgate fiquem às cegas.

---

## 🚀 Funcionalidades e Diferenciais
* 💾 **Retenção Local Segura (Zero-Loss):** Capacidade de manter leituras meteorológicas íntegras por até 72 horas em total isolamento da rede.
* 📡 **Comunicação de Contingência:** Disparo de pacotes compactados de alerta via sinais de rádio UHF quando as torres de telefonia caem.
* 🔄 **Sincronização Assíncrona Inteligente:** Descarregamento cronológico automatizado (FIFO) da base de dados assim que a estabilidade TCP/IP é retomada.
* ♿ **Interface Institucional Clean UI:** Front-end otimizado para legibilidade absoluta, sem a dependência de JavaScript, implementado com semântica HTML5 avançada e suporte a leitores de tela (a11y).

## 📂 Arquitetura do Portal
O projeto web institucional está estruturado para mapear de forma clara e corporativa todas as facetas da solução:
1. **Página Inicial (`index.html`):** Visão macro do sistema e atualizações operacionais.
2. **O Problema (`problema.html`):** Análise do colapso estrutural das comunicações atuais em crises climáticas.
3. **Tecnologias (`tecnologia.html`):** Especificações do hardware embarcado, protocolos de transmissão e armazenamento.
4. **Objetivos e Público (`objetivos.html`):** Direcionamento estratégico, impacto na Defesa Civil e proteção populacional.
5. **Benefícios e Prática (`beneficios.html`):** Vantagens logísticas no socorro a vítimas e operação cotidiana.
6. **Auditoria (`logs.html`):** Terminal de rastreio de eventos cibernéticos, timeouts e reconexões do sistema.

## 🛠️ Como Executar o Projeto Localmente
O portal foi desenvolvido visando máxima compatibilidade e leveza, utilizando marcação pura. Não há necessidade de instalação de dependências locais (Node.js, NPM, etc.).

1. Efetue o clone deste repositório para sua máquina local ou baixe o arquivo em `.zip`.
2. Assegure-se de que a folha de estilos (`style.css`) e os ativos visuais (`.jpeg`) estejam na mesma raiz dos arquivos de documentação (`.html`).
3. Dê um duplo clique no arquivo `index.html` para iniciar a navegação em seu navegador web padrão.
